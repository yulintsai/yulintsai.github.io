[Day 29] Oops！Block DevOps

Oct 7, 2020 · 1 min read

How to BLOCK DevOps？

要如何阻礙DevOps？ DevOps是一種文化，那就只要讓該文化導向毀滅之路就可以了。 以下介紹四點可能可以BLOCK DevOps的方式

不 - 溝通

溝通是DevOps裡面，蠻重要的技能，如何跟他人溝通？ 在溝通之前，請遵守一個原則，尊重他人，ex:可以把「麻煩你」+「謝謝」掛在嘴邊。

那麼 How to BLOCK DevOps？

  • 常常溝通不尊重他人時，詢問問題得到回答後，直接沈默。
  • 明明已經看到問題正在處理卻不先回對方讓對方等待時。
  • 溝通時不回應、不想理或回答一半時。
  • 常常文不對題或問A答B時。
  • 很少接受他人的建議時。
  • 出問題時怪東怪西怪天怪地怪機器。
  • 別人在開會中提出的問題，永遠沈入海底。

不 - 思考

這裡的思考主要是能不能透過你站在對多個人的架構上來思考事情，主要是換位思考、團隊思考。

那麼 How to BLOCK DevOps？

  • 當有人有問題時，無法自行思考。
  • 當你給人說某時間給你回覆時卻沒回覆，造成他人無限等待時。
  • 當你無法做決定卻不去找人溝通時。
  • 當你考慮的事務很多卡住，卻不找人溝通時。
  • 我永遠只考慮我個人，無法破框，不從團隊、公司角度考慮問題時。

不 - 透明度

每個事情盡量都是透明的，在討論議題時，不管是誰都能參與，可以接受提議以及討論。

那麼 How to BLOCK DevOps？

  • 當你急著命令別人工作怎麼做卻不告訴別人原因時。
  • 當你跟同事在同一個團隊，卻不知道整個團隊在做什麼時。
  • 當你從別人口中聽到全團隊只有沒告訴你的議題，但你有更好的解決方案，此時卻因為議題不透明所以不想講時。

不 - 自動化

不要做重複性質的工作，若能把需求SOP化就能把工作自動化，很多CI/CD工具能幫你解決。

那麼 How to BLOCK DevOps？

  • 每天早上要執行某某作業流程，明明可以自動化，卻不做自動化，造成每天早上你永遠都在做那些機器人可以做完的事情時。
  • 每個月要產生報表，明明可以自動化，卻不自動化，造成月底為了做報表在忙著處理月結報表時。
  • 明明每次要推code要跑檢查可以利用CI/CD工具自動跑 test case 卻每次都要手動執行一堆指令時。

誰說DevOps只有一個團隊能實現，多個團隊也能夠順暢的DevOps。

那麼 How to BLOCK 多個團隊的 DevOps？

  • 不主動追需求、追問題。
  • 怕別人一直問、怕一直追進度。
  • 在團隊內建立多個小團體，讓多個小團體產生裂痕。
  • 提供的文件常常有誤對照不了。
  • 需求的使用情境，雙方無法認知。
  • 其他團隊不想懂或者根本聽不懂你們的需求。

小結

Oops 看完這麼多 How to BLOCK DevOps是不是覺得DevOps的實現真的要注意很多事情呢？
